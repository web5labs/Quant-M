# Useful Patterns: local-copy:quantm/The-Sataff/Ponboarding

## Patterns to borrow

- Onboarding before implementation.
- Explicit project rails: spec, shippable definition, assumptions, open questions, validation plan.
- Wiki-first context reduction before large implementation slices.
- Goal generation and linting as durable controls for future agent sessions.

## Patterns to avoid copying blindly

- Do not let onboarding rails become a substitute for actual runtime validation.
- Do not keep placeholder planning text once Quant-M-specific evidence exists.
- Do not widen the spec into product fantasies that the local repo does not support.

## Relevance to current project

Ponboarding is directly relevant because Quant-M is being prepared as a real agent-built project. Its strongest value is keeping future implementation slices bounded, reviewable, and evidence-backed.
