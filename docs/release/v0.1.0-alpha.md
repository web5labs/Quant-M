# Quant-M v0.1.0-alpha

Quant-M is a local-first Rust control plane and flight recorder for AI-assisted work. This alpha release focuses on evidence, replay, capability truth, FSM authority, side-effect safety, and clean continuation between agents or models.

Release posture: public developer preview.

## Highlights

- Added FSM authority inspection with `quant-m fsm authority`.
- Added capability truth reporting so wired, partial, audited, and design-only features are easier to distinguish.
- Added typed FSMs for worker jobs, skill execution, Context Guardian, and workflow cursor ordering.
- Added a shared side-effect policy gate for high-risk runtime paths.
- Hardened shell-backed skill execution so blocked actions are recorded as blocked, not failed.
- Preserved safe defaults: no hidden provider calls, no automatic shell execution, no channel-as-authority, and no live trading.
- Improved context continuity with typed Context Guardian state and recommended actions.
- Cleaned planning and strategy docs to reduce repeated prose and avoid Markdown drift.

## Safety Posture

Quant-M is safe by default. Workers propose; the governed core decides. Provider calls, shell execution, network sends, Telegram/webhook paths, and other side-effecting actions are gated. Replay does not repeat side effects. Trading-related surfaces remain paper-only, dry-run, blocked, denied, or unavailable.

## Notes

This is an alpha/developer-preview release. It is intended to demonstrate Quant-M's local-first governance model, replay discipline, and runtime safety boundaries. It is not a production trading system and does not perform live trading.
